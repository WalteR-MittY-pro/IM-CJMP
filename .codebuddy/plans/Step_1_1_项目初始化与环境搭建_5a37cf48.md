## 产品概述

完成 Telegram CJMP 项目的 Step 1.1 项目初始化与环境搭建，确保项目编译通过、可安装运行、显示空白页面无崩溃。

## 核心功能

- 将应用名称从 "telegram_cjmp" 修改为 "Telegram"
- 将入口页面从按钮改为空白页面（Column 组件），为后续页面开发做准备
- 验证项目编译和运行状态

## 技术栈

- 开发语言：仓颉（Cangjie）
- UI 框架：Cangjie UI（基于 ohos.component）
- 目标平台：HarmonyOS（hap）

## 实现方案

基于现有 CJMP app 工程结构，进行最小化修改：

1. **应用名称配置**：修改两处 string.json 资源文件，统一应用显示名称为 "Telegram"
2. **入口页面**：保留基本的页面结构（Row + Column 布局），移除按钮组件，实现空白页面效果

## 目录结构

```
telegram_cjmp/

│   │   └── resources/base/element/

│   └── entry/src/main/resources/base/element/
│       └── string.json                   # [MODIFY] 修改 EntryAbility_label 为 "Telegram"
```

## 实现说明

### index.cj 修改要点

- 移除 `@State var message` 状态变量（空白页面不需要）
- 保留 `Row` 和 `Column` 基础布局结构
- 移除 `Button` 组件及其点击事件
- 保持 `width(100.percent)` 和 `height(100.percent)` 确保页面占满屏幕

### 构建与验证命令

```
# 构建 HarmonyOS 应用
keels build hap

# 运行到设备
keels run
```
