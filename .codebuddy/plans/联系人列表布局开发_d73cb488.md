## 用户需求

在telegram_cjmp工程目录中，按照cjmp项目工程规则和仓颉UI语法，实现5.1联系人列表布局组件开发

## 产品概述

实现联系人列表页面，包含导航栏、搜索栏、联系人列表和分组索引

## 核心功能

- 导航栏：显示"联系人"标题，右上角添加按钮(+)
- 搜索栏：包含搜索图标和输入框
- 联系人列表：显示联系人头像、名称、用户名和在线状态
- 分组索引：按字母分组显示（A-Z）
- 组件规范：头像48x48dp，列表项高度64dp

## Mock数据格式

```
[
  {"name": "Alice", "username": "alice", "online": true},
  {"name": "Bob", "username": "bob", "online": false, "lastSeen": "5分钟前"}
]
```

## 验证方法

- [ ] 显示联系人列表
- [ ] 显示头像、名称、在线状态

## 技术栈

- 框架：Cangjie + ArkUI
- 语言：Cangjie
- 组件：List, ListItem, Row, Column, Text, TextInput

## 技术架构

### 系统架构

- 采用与ChatsPage.cj相同的架构模式
- Component组件声明方式
- @State管理状态

### 模块划分

- **MockData模块**：添加ContactItem数据结构和联系人数据
- **ContactsPage模块**：联系人列表页面组件

### 数据流

- 页面加载 → 获取MockData → 渲染联系人列表 → 用户交互

## 实现方案

1. 在MockData.cj中添加ContactItem结构体和联系人数据数组
2. 修改ContactsPage.cj参考ChatsPage.cj布局模式实现：

- 导航栏Row包含标题和添加按钮
- 搜索栏Row包含搜索图标和TextInput
- 联系人List使用ForEach遍历contactList
- 列表项包含头像(48x48)、名称、@username、在线状态
- 分组索引显示在联系人前面

3. 列表项高度设置为64dp
