# Telegram CJMP MVP UI 设计文档

## 版本：1.0
## 目标平台：Android / iOS / HarmonyOS
## 文档类型：UI 功能接口设计

---

## 一、设计原则

### 1.1 MVP 定义
- **最小可行产品**：仅包含用户最核心的聊天功能
- **功能优先级**：聊天 > 联系人 > 基础设置
- **界面简洁**：去除非核心的装饰性元素

### 1.2 核心功能范围
| 模块 | MVP 功能 |
|------|---------|
| 聊天列表 | 显示、进入聊天 |
| 聊天界面 | 收发文本消息 |
| 联系人 | 联系人列表、发起聊天 |
| 基础设置 | 主题切换、深色模式 |

---

## 二、界面结构

### 2.1 页面层级

```
主界面（聊天列表）
    │
    ├── 点击列表项 ──→ 聊天详情页
    │
    ├── 点击菜单图标 ──→ 侧边栏菜单（抽屉导航）
    │       │
    │       ├── 主页 ──→ 返回聊天列表
    │       ├── 联系人 ──→ 联系人列表
    │       ├── 设置 ──→ 设置页面
    │       └── ...
    │
    └── 底部 Tab 切换
            │
            ├── Contacts ──→ 联系人列表
            ├── Chats ──→ 聊天列表（当前）
            └── Settings ──→ 触发侧边栏菜单
```

### 2.2 页面导航

| 页面 | 入口 | 出口 |
|------|------|------|
| 聊天列表 | 默认主页 / Tab Chats | 点击进入聊天 / 打开侧边栏 |
| 聊天详情 | 聊天列表点击 | 返回、发送消息 |
| 侧边栏菜单 | 导航栏菜单图标 / Tab Settings | 点击菜单项 / 点击空白区 |
| 联系人列表 | 侧边栏菜单 / Tab Contacts | 返回、发起聊天 |
| 设置页面 | 侧边栏菜单 | 返回 |

- 占位符："Search for messages or users"

- 圆角：8dp
- 左侧搜索图标：灰色

**列表项**
- 高度：80dp
- 头像：48x48dp，圆形，左侧对齐
- 名称：主标题，16sp，黑色，最多一行
- 最后消息：副标题，14sp，灰色，最多两行

- 背景色：白色

- 未选中状态：图标和文字为灰色

**间距规范**
- 列表项左右内边距：16dp
- 头像与文字间距：12dp
- 文字区域内边距：8dp

## 四、接口设计

### 4.1 接口设计原则

- **RESTful 风格**：资源 + 操作
- **数据格式**：JSON
- **接口分类**：数据接口 / 业务接口

### 4.2 聊天模块接口

#### 4.2.1 获取聊天列表

**接口路径**
```
GET /api/v1/chats
```

**请求参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页数量，默认 20 |

**响应数据结构**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "chats": [
      {
        "id": "chat_123",
        "type": "private",
        "name": "用户名",
        "avatar_url": "https://...",
        "last_message": {
          "content": "最后消息内容",
          "timestamp": 1706789000,
          "is_unread": true,
          "unread_count": 3
        }
      }
    ],
    "total": 100,
    "page": 1,
    "has_more": true
  }
}
```

#### 4.2.2 获取聊天消息

**接口路径**
```
GET /api/v1/chats/{chat_id}/messages
```

**路径参数**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| chat_id | string | 聊天 ID |

**请求参数**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| before_id | string | 否 | 消息 ID（用于分页） |
| limit | int | 否 | 数量限制，默认 50 |

**响应数据结构**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "messages": [
      {
        "id": "msg_001",
        "chat_id": "chat_123",
        "sender_id": "user_456",
        "type": "text",
        "content": "消息内容",
        "timestamp": 1706789000,
        "status": "sent"
      }
    ],
    "has_more": true
  }
}
```

#### 4.2.3 发送消息

**接口路径**
```
POST /api/v1/chats/{chat_id}/messages
```

**路径参数**
| 参数名 | 类型 | 说明 |
|--------|------|------|
| chat_id | string | 聊天 ID |

**请求体**
```json
{
  "type": "text",
  "content": "消息内容"
}
```

**响应数据结构**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "message": {
      "id": "msg_002",
      "chat_id": "chat_123",
      "sender_id": "user_789",
      "type": "text",
      "content": "消息内容",
      "timestamp": 1706789100,
      "status": "sending"
    }
  }
}
```
